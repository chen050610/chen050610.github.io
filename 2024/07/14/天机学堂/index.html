使用的技术栈

  • Maven

    • 聚合 父工程通过modules标签，将子模块聚合起来，父工程执行maven指令，所有的子模块也会执行
    • 继承 子模块通过parent标签 可以从父工程继承一些依赖
    • clean 清理
    • default （complie,test,package,install）
    • site (deploy 部署)
    • 上面三套的生命周期相互独立，但是在同一个生命周期的内部，执行后面的指令，后一次执行前面的指令

  • nacos服务注册时候相注意一下问题

    • 因为我们在开发环境，使用nacosip设置为本地的ip即可，注意是.1的而不是101，因为101是虚拟机的ip地址，注意这里不能写localhost，如果我们有远程调用的话，已经部署在虚拟里的服务远程调用，如果写localhost访问的就是虚拟机里的

  • 部署的微服务不在本地，我们可以利用IDEA的远程调试的功能

    • Long id = 12L;
这一段代码就会发生自动装箱，调用valueOf的方法，如果字段在这个区间的话，就会直接返回缓存否则创建一个新的Long对象，==比较的是地址
