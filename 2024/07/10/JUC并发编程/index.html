java线程理解以及openJdk中的实现

java多线程的相关概念

  • 1把锁，2个并（并发【concurrent】，并行【parallel】），3个程（进程，线程，管程）
    • 并发：同一时刻只有一件事情在做，一台服务器做多个任务
    • 并行：多台处理器同时处理多个任务，
    • 管程：就是平常说的锁对象,Monitor监视器

  • 用户线程和守护线程

    • 一般就是用户线程
    • 用户线程 业务的线程
    • 守护线程 例如GC垃圾回收线程，服务线程

  • 线程的Daemon属性

    • 返回true 守护线程
    • false 用户线程

不做任何的配置，new出来的线程就是用户线程，用户线程结束，守护线程也会结束

  • CompletableFuture
    • Future接口，定义操作异步任务的执行的一些方法，获取异步任务执行的结果，判断任务是否完成，是否被取消，取消任务的执行

查看的顶层的源码，Thread的构造方法不能接受Callable接口，我们需要实现三个要求，1.能够执行异步任务，2.异步任务有返回值，3.或者异步任务的信息，前一个我们的Runnable接口可以实现，后两个Future接口可以实现，这时候就可以考虑我们的设计模式，我们的FutureTask实现了RunnableFuture接口

，然后在FutureTask里面构造注入Callable

  • Future 的缺点
    • get方法容易堵塞
    • Future对于结果的获取，不是很友好，只能通过阻塞或者轮询的方式得到任务的结果

第一个代码片段

在这个代码中，首先通过ids.stream().map(...)为每个id生成一个CompletableFuture对象，这些对象被收集到一个列表中。此时，所有的CompletableFuture任务都已经被提交到异步执行环境中（如默认的ForkJoinPool），但它们的执行是异步的，即collect(Collectors.toList())时，这些任务可能还没有完成。

然后，对这个包含CompletableFuture的列表进行第二次流操作，通过map(CompletableFuture::join)等待每个CompletableFuture的结果。这里，join方法会阻塞当前线程直到对应的CompletableFuture完成。

第二个代码片段

在这个代码中，流操作是连续的，且map(CompletableFuture::join)直接在第一个map操作之后执行。这意味着，对于流中的每个元素，都会立即等待对应的CompletableFuture完成，然后再继续处理下一个元素。这会导致流操作在某种程度上顺序化，因为每个join调用都会阻塞直到其对应的异步操作完成。

耗时差异的原因

  1. 并行性：在第一个代码片段中，所有的CompletableFuture任务几乎是同时提交的，这充分利用了异步执行的并行性。而第二个代码片段中，由于join的阻塞性质，它实际上限制了并行处理的能力，因为每个id的处理都需要等待前一个id的结果。
  2. 资源利用：第一个代码片段中，由于所有任务几乎同时启动，它们可以更好地利用可用的处理器资源。而在第二个代码片段中，由于join的串行化效果，资源利用可能不那么高效。
  3. 线程阻塞：第二个代码片段中的join调用会导致线程阻塞，这减少了程序在等待异步操作完成时的并行性

Java的锁

  • 乐观锁
  • 悲观锁

静态方法synchronzied的锁对象是当前的类对象，普通方法是当前的this

  • 面试题： 为什么认为的一个对象，都能成为锁

每一个对象天生都带有一个对象监视器，每一个被锁对的对象都会和monitor关联起来

  • 公平锁和非公平锁，默认的是非公平锁，因为非公平锁能够充分的利用cpu的资源，不用等待其他的线程的完成，和线程的切换

    • 非公平锁 抢夺的锁的概率不相同，

    • 公平锁 多个线程按照顺序

  • 可重入锁（递归锁）

    • 指同一个线程在外层获取锁的时候，再次进入该线程的内层方法会自动获取锁（前提，锁对象是同一个），不会因为之前已经获取过没有释放而堵塞

    • 每一个锁对象有一个锁计数器和指向持有该锁的的线程的指针，为了方便以后释放锁，当进入一个锁以后计数器就会加1，离开锁以后就会减一

    • 分为两种

      • 隐式锁（synchronized关键字使用的锁）默认的是可重入锁，

      •         final Object object = new Object();

        new Thread(() -> {
            synchronized (object){
                System.out.println(Thread.currentThread().getName()+"\t ----外层调用");
                synchronized (object){
                    System.out.println(Thread.currentThread().getName()+"\t ----中层调用");
                    synchronized (object){
                        System.out.println(Thread.currentThread().getName()+"\t ----内层调用");
                    }
                }
            }
        },"t1").start();
